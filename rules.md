
## 2. การอัปเดตโค้ด
- **ดึงโค้ดล่าสุดเสมอ**: เมื่อเริ่มทำงานใหม่หรือมีการสลับงาน จะต้องทำคำสั่ง git pull origin main (หรือดึงข้อมูลล่าสุด) ทุกครั้ง เพื่อป้องกันปัญหาการแก้ไขไฟล์ชนกัน (Conflict) กับเพื่อนร่วมทีม

## 3. การตรวจสอบคุณภาพงาน (Quality Assurance)
- **Self-Review ก่อนส่งงาน**: เมื่อทำงานที่ได้รับมอบหมายเสร็จแล้ว **ห้าม** แจ้งว่าเสร็จทันที แต่จะต้องสวมบทบาทเป็น "ผู้ตรวจงาน (Reviewer)" เพื่อตรวจสอบโค้ดที่ตัวเองเขียน (ค้นหาบั๊ก, ตรวจสอบ Responsive, เช็คผลกระทบกับไฟล์อื่น) และปรึกษา/รายงานผลการตรวจให้ผู้ใช้ทราบก่อนเสมอ เพื่อลดข้อผิดพลาดให้เหลือน้อยที่สุด

## 4. การแก้ปัญหาและแก้ไขบั๊ก (Problem Solving & Debugging)
- **พิสูจน์ก่อนแก้ (Verify, Don't Guess)**: เมื่อพบปัญหาหรือได้รับแจ้งให้แก้บั๊ก **ห้ามเดาคำตอบโดยเด็ดขาด** จะต้องใช้เครื่องมือที่มี (เช่น ค้นหาโค้ด, รันทดสอบ, ตรวจสอบไฟล์, หรือรันสคริปต์จำลอง) เพื่อ **ยืนยัน (Verify)** ให้เห็นหลักฐานของปัญหาด้วยตาตัวเองก่อน แล้วจึงค่อยเสนอวิธีแก้ไข
